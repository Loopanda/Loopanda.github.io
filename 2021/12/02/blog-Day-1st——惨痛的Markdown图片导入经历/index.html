Hexo图片加载问题

首先我要说这个问题呢并没有完全解决，就算是解决了一半吧

分享一些关于我“成果”的经验(基于Win11操作系统)

首先在md文件中本地图片的导入方式最好为

![](./img/1.png)

采用相对路径的话，可以保证登录网页的时候不会产生路径间的冲突

  • 首先在cmd上执行以下指令

    npm remove hexo-render-pug --save
npm add https://github.com/CodeFalling/hexo-asset-image
start -config.yml

  • 在弹出的界面窗口中将post_asset_folder修改为true：

  post_asset_folder:true

然后应该md文件中使用相对路径在进行

hexo g

就可产生发现类似于如下的提示

update link as:-->/2021/12/02/Markdown/123321.png
update link as:-->/2021/12/02/Markdown/123321.png

表示同名文件下的index.html文件中已经包含或者通过某种手段连接了该图片
便可在初始化html的时候找到该文件（这也是给我挖的一个大坑）如下所示

然后进行操作：

hexo s

打开本地浏览器进行调试

但是我用这个方法进入localhost:4000后还是给我报了错

在本地 __localhost:4000中查找到的错误是index文件并没有和png文件link__（ 这里是一个 天坑 ）
结果图片还是无法显示出现了下图的情况

于是乎我就想既然是 404 Not Found 那我把图片加入到index对应目录不就行了吗
然后就出现了手动拖拽图片到如下同名目录的操作

总算这样做结果是好的，在我的localhost:4000下终于成功显示了图片
但是我并不开心
因为我知道
虽然我这样做了结果是好的符合了我最开始的要求，但这种方法无疑是很笨的，如果每次都要手动去拖拽的话一个是一点也 不优雅 ，还有就是 post_asset_folder就失去了他原有的价值 ，所以我开始考虑着手修改源码，就是index.js文件，但是此时的我又没学过JavaScript所以看起来十分费劲，修改了几个split点后发现，效果还没有原来的好，host g的时候老是fail，当我想了一晚上也没得出结论的时候我决定 开摆！ 然后正当我一点也不甘心的写这篇博文的时候，我想算了就算是个错的我也要把它部署到我的github仓库里面当一个经验教训好好研究，于是乎我删除了 123321.png 但是当我打开 Loopand.github.io 的时候，图片！他！惊人的出现了！！！ohhh 所以不得不说这是一个天坑，在此我留下两个疑问，也是我以后努力要去寻找答案的点

  1. 为什么部署到远端的时候link达成而localhost则失效
  2. 远端是以什么形式link我的代码的

其实说实话我的心里已经有了几个想法

  • 本地的localhost可能通过license无法达成link而github仓库或者说其他服务器可以
  • 可能和post_asset_folder中的index.js有关，如果我将js代码修改成在同名html文件目录下复制粘贴source/img下的所有图片，可能可以达成相同的效果
  • 最后对自己说一下github看代码一定要看README，太重要了可以省好多事，然后这里的相对路径一定要是 . / 开头

坑就下次再填了，我已经从20:00调试这个调试到了2:00，总体来说这个问题看似简单其实好像又不是那么简单，希望以后经过我对JavaScript和License的深度学习我能轻松地回答这个问题吧。
